Task Description:
As part of my programming and system design learning journey, I independently performed four key tasks that contribute significantly to understanding real-world applications of core computer science concepts. These include a File Handling Program, Linked List Implementation, Lexical Analyzer (Compiler Design), and a Data Compression Tool using Run-Length Encoding (RLE). Each of these tasks strengthened my practical skills in C programming and problem-solving and provided hands-on experience with file systems, data structures, compiler theory, and data compression algorithms.

1. File Handling Program

Task Overview:
This program demonstrates how to create, read, write, and append data to files using C. It simulates real-world scenarios like logging, report generation, or text processing by manipulating plain text files.

Tools & Platforms Used:
Programming Language: C
IDE: VS Code
Platform: Windows and Linux (Ubuntu)

Purpose & Applicability:
File handling is fundamental to almost all applications. This task is applicable in developing database systems, logging mechanisms, student record management systems, and text editors. It teaches file stream management, error handling, and data persistence — skills essential for backend developers and systems programmers.


2. Linked List Implementation

Task Overview:
This task involved implementing a singly linked list with operations such as insertion, deletion, and traversal. The objective was to understand dynamic memory allocation and pointer-based data structure manipulation.

Tools & Platforms Used:
Language: C
IDE: VS Code
Platform: Windows & Linux (CLI environment)

Purpose & Applicability:
Linked lists are a cornerstone of data structures and are widely used in memory management, real-time systems, and creating custom data types like stacks and queues. This task is vital in understanding how memory is managed dynamically and is applicable in OS development, embedded systems, and complex algorithm design.


3. Compiler Design: Lexical Analyzer

Task Overview:
I created a simple lexical analyzer that reads a source file and identifies tokens such as keywords, operators, and identifiers. This is the first phase of a compiler, essential for source code parsing.

Tools & Platforms Used:
Language: C
Editor: VS Code
Platform: Windows/Linux
Input medium: .c source file

Purpose & Applicability:
Lexical analyzers are foundational to compiler design and interpreters. This task is applicable in creating custom scripting languages, code editors with syntax highlighting, and advanced development environments. It helps in understanding how programming languages are interpreted at the lowest level.


4. Data Compression Tool (RLE)

Task Overview:
This tool performs basic data compression and decompression using Run-Length Encoding. The focus was on reading from and writing to files while applying simple yet effective data transformation logic.

Tools & Platforms Used:
Language: C
Compiler: VS Code 
Platform: Cross-platform (Tested on Windows and Linux)

Purpose & Applicability:
Compression tools are used to reduce storage space and improve data transfer efficiency. This RLE-based tool is useful for compressing repetitive data, such as logs, simple images, and texts. It lays the groundwork for understanding more complex algorithms like Huffman and LZW and is applicable in data storage systems, networking, and file archivers.
